# SoK: Generative AI in Assessment — Engineering, STEM & CS Education

Systematization of Knowledge (SoK) review repository.  
**Target venue:** IEEE TALE 2026 — Assessment Revolution track.

---

## Overview

This repository contains the full dual-pipeline data extraction and analysis workflow for a Systematization of Knowledge (SoK) review that maps, synthesises, and assesses the evidence quality of research on Generative AI in educational assessment within engineering, STEM, and computing education (2022–mid-2026).

Unlike a conventional systematic review, this SoK explicitly characterises the **shape** of the knowledge base — thematic clusters, methodological distribution, evidence quality per category, and structural gaps — rather than synthesising around a single pre-specified research question.

**Final corpus:** 641 papers | **PDF coverage:** 617/641 (96.3%)  
**Bibliometric dataset:** 2,122 records (5 databases, deduplicated)  
**Search date:** June 2026

## Dual-Pipeline Analysis

This SoK uses two parallel, independent pipelines. Where both converge, findings gain confirmatory weight; where they diverge, areas of nuance are identified.

### Pipeline 1 — Corpus Construction & OA Discovery

```bash
# 1. Build corpus from BIB exports
python3 pipeline1_retrieval/corpus_builder.py

# 2. Screen all records (title + abstract)
python3 pipeline1_retrieval/screen.py

# 3. Discover open-access PDF URLs (Semantic Scholar, OpenAlex, Unpaywall)
python3 pipeline1_retrieval/find_oa_all.py --email your@email.com

# 4. Download OA papers using discovered URLs in find_oa_report.json.
#    Retrieve paywalled papers through your institution's database access.

# 5. Verify downloaded PDFs
python3 pipeline1_retrieval/verify_pdfs.py
```

**PDF retrieval note:** This repository provides the corpus manifest and OA URL discovery tooling. Researchers reproducing this corpus should retrieve paywalled papers through their own institutional subscriptions.

### Pipeline 2 — SoK Evidence Synthesis

#### Step A: Bibliometric analysis (R)

IEEE Xplore, ACM, and ERIC BibTeX exports are first normalised to Scopus-compatible format, then merged with Scopus and WoS records. Author affiliations absent from IEEE/ACM/ERIC exports are retrospectively enriched via the OpenAlex API (96.9% coverage achieved).

```bash
# 1. Normalise IEEE/ACM/ERIC BibTeX to Scopus-compatible format
python3 pipeline2_analysis/bibliometrics/00_normalize_bib.py
# → data/raw/ieee_acm_eric_normalized.bib

# 2. Load all 5 databases, merge, deduplicate (2,122 records)
Rscript pipeline2_analysis/bibliometrics/01_load_and_merge.R
# → data/processed/bibliometrix_merged.rds

# 3. Export RDS to CSV for affiliation enrichment
Rscript -e 'write.csv(readRDS("data/processed/bibliometrix_merged.rds"),
            "data/processed/bibliometrix_merged.csv", row.names=FALSE)'

# 4. Enrich affiliations for IEEE/ACM/ERIC records via OpenAlex (resumable)
python3 pipeline2_analysis/bibliometrics/01b_enrich_affiliations.py
# → data/processed/affiliations_enriched.csv
# Note: script checkpoints every 50 records — safe to interrupt and re-run

# 5. Apply enriched affiliations back to RDS
Rscript pipeline2_analysis/bibliometrics/01c_apply_affiliations.R

# 6. Generate all figures and tables
Rscript pipeline2_analysis/bibliometrics/02_bibliometric_analysis.R
# → pipeline2_analysis/outputs/figures/*.png
# → pipeline2_analysis/outputs/tables/*.csv
```

#### Step B: Full-text content analysis (Python)

```bash
# Requires PDF corpus in _pdfs/
pip install pymupdf tqdm
python3 pipeline2_analysis/content_analysis/01_text_extraction.py
python3 pipeline2_analysis/content_analysis/02_thematic_coding.py
# → pipeline2_analysis/outputs/tables/taxonomy_summary.csv
# → pipeline2_analysis/outputs/tables/thematic_coding.csv
```

## Seven-Category SoK Taxonomy

The SoK synthesises the corpus across seven thematic categories derived from full-text analysis of 617 papers:

| Cat. | Theme | Papers | % | Evidence Quality |
|---|---|---|---|---|
| A | Automated Grading | 290 | 47.0% | Moderate — small-n empirical, limited validity |
| B | Authentic Assessment Redesign | 277 | 44.9% | Low — conceptual/framework, few evaluations |
| C | Academic Integrity | 430 | 69.7% | Moderate — detection studies; policy surveys |
| D | Formative and Adaptive Assessment | 439 | 71.2% | Moderate-High — controlled studies in ITS/CS1 |
| E | Ethics and Policy | 557 | 90.3% | Low — framework/opinion; sparse empirical |
| F | Perception and Affect | 572 | 92.7% | Moderate — survey-heavy, cross-sectional |
| G | Review / Methodology | 577 | 93.5% | N/A — methodological marker |

Categories are not mutually exclusive.

## Key Findings

- **Growth**: 2,122 records; CAGR 296.6% (2022–2025); ~9× growth 2023–2025
- **Venues**: 777 unique sources; ASEE (61), ACM SIGCSE (92 combined), FIE (43) dominant
- **Authors**: Denny P (51), Leinonen J (35), MacNeil S (24), Prather J (21)
- **Geography**: US (18.8%), China (15.2%), India (5.0%); 9.9% international co-authorship after affiliation enrichment
- **Keyword clusters**: Disciplinary Contexts (1,681), GenAI/Assessment Discourse (1,662), LLM Applications (1,417), CS/Computing Pedagogy (987)
- **Structural gap**: Diagnostic/normative work dominates; controlled evaluations of implemented interventions <10% of corpus
